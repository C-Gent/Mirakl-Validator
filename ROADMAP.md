# Mirakl Validator Roadmap

This roadmap outlines the planned evolution of the **Mirakl Hierarchy Validator** into a complete Mirakl Import Validation Suite.  
It includes UI enhancements, new validators, file generators, API tooling, and QA automation features.

The roadmap is versioned and structured to support long‑term growth.

---

# 📌 Current Phase  
**v1.2.x — UI Modernisation & Component Architecture**

The project now uses a modular ES‑module architecture with:

- Drag‑and‑drop upload zone  
- File information panel  
- Validation summary banner  
- Statistics cards  
- Categorized collapsible error viewer  
- Two‑panel responsive layout  
- Component‑based rendering  
- Structured validation pipeline  

This foundation enables the next major phases of development.

---

# 🚀 Phase 1 — v1.3.x  
## Exportable Validation Reports

### Goals
- Allow users to export validation results for offline review or sharing.
- Support multiple formats.

### Planned Features
- Export errors as:
  - CSV
  - JSON
  - HTML report
- Include:
  - Summary banner
  - Stats
  - Categorized errors
  - Timestamp
  - File metadata

### Dependencies
- Finalised categorized error structure (completed in v1.2.6)
- Stable UI layout (completed in v1.2.0)

---

# 🚀 Phase 2 — v1.4.x  
## Mirakl Attribute Validator

### Goals
Validate Mirakl attribute import files using the same UI and validation engine.

### Planned Features
- Attribute CSV validator
- Required field checks
- Type validation (string, number, boolean, list)
- Allowed values validation
- Attribute group validation
- Duplicate attribute detection
- Parent attribute relationships
- Categorized error viewer (reuse v1.2 component)

### Dependencies
- Extend validator.js to support multi‑schema validation
- Add schema registry for Mirakl file types

---

# 🚀 Phase 3 — v1.5.x  
## Mirakl Value Validator

### Goals
Validate Mirakl value import files (attribute values).

### Planned Features
- Value CSV validator
- Attribute‑value mapping validation
- Duplicate value detection
- Missing attribute detection
- HTML/formula injection protection
- Categorized error viewer

### Dependencies
- Attribute validator (v1.4.x)
- Schema registry

---

# 🚀 Phase 4 — v1.6.x  
## API Response Validator

### Goals
Validate responses returned by Mirakl API endpoints.

### Planned Features
- Upload JSON response files
- Validate structure against Mirakl documentation
- Highlight missing fields, unexpected fields, type mismatches
- Categorized error viewer for API responses
- Support:
  - Success responses
  - Error responses
  - Warning responses

### Dependencies
- JSON parsing module
- Mirakl API schema registry

---

# 🚀 Phase 5 — v1.7.x  
## API Error Report Validator

### Goals
Validate and interpret error reports generated by Mirakl import endpoints.

### Planned Features
- Upload Mirakl error report CSVs
- Categorize errors:
  - Missing fields
  - Invalid values
  - Unknown codes
  - Parent/child mismatches
  - Structural errors
- Provide human‑readable explanations
- Suggest fixes

### Dependencies
- API response validator (v1.6.x)
- Error category engine (v1.2.x)

---

# 🚀 Phase 6 — v1.8.x  
## Interface File Generator (Flat File Builder)

### Goals
Generate interface files from API responses.

### Planned Features
- Upload API response JSON
- Convert to flat file format:
  - Reference
  - Key
  - One‑line API response per record
- Export as:
  - CSV
  - XLSX
- Mapping templates for:
  - Categories
  - Attributes
  - Values
  - Offers
  - Products

### Dependencies
- API response validator (v1.6.x)
- File export module
